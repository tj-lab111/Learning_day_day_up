工具生态定位：Dify vs AI编程 vs Agent

Dify Workflow, AI 编程工具与智能体 (Agent) 的差异化定位

🏗️ Dify Workflow

  • 本质：基于 LLM 的低代码业务编排。
  • 特点：结构化强、稳定性高、易于复用。
  • 适用：SOP 固定的企业级业务流（如：知识库问答、自动化报表）。

💻 AI 编程工具

  • 本质：Cursor/Claude Code/Trae 等辅助编码 IDE。
  • 特点：灵活、无上限、需代码基础。
  • 适用：开发 Dify 无法实现的特化插件或复杂算法节点。

🤖 智能体 (Agent)

  • 本质：具备自主规划能力的数字员工。
  • 特点：灵活但不可控，适合模糊任务。
  • 适用：需要自主决策的场景（如：去网上找最便宜的机票）。

为什么选择低代码平台落地？

📜 结构化即文档

画布就是地图。解决“人走茶凉、逻辑失传”的问题，降低交接运维成本。

🎨 非技人员运营

纯图形化界面，让最懂业务的人直接改 Prompt 和测效果，不用写代码，也不用等开发排期。

🏭 基建全封装

自带 API、日志、监控。直接提供“拎包入住”的工厂，省去 90% 后端搭建工作。

🔄 数据流闭环

实现“结果即时标注”。点点气泡就能优化 AI 回答，把“反馈-修正”的路径缩到最短。

CHAPTER 01

基础知识：Dify 节点全解析

实现了“业务逻辑”与“底层代码”的解耦。

核心底座

1. 基础大脑节点

  • LLM (模型)：一切推理的核心。 场景：输入会议速记原始文本，要求 AI 提炼出 5 条核心决议。
  • 知识检索 (Recall)：让 AI 拥有“长短期记忆”。 场景：用户询问“公司报销标准”，节点自动去《行政手册.pdf》中精准定位对应条文。
  • 直接回复：最简单的沟通节点。 场景：在工作流开头或结尾，直接给用户一个友好的“正在为您分析中，请稍后...”提示。
  • Agent (智能体)：具备自主性的“数字员工”。 场景：完成“帮我订一张去上海最便宜的机票”这种需要查票、选价、下单多步操作的复杂任务。
流程控制

2. 逻辑枢纽节点

  • 问题分类器：识别意图。 场景：判断用户的输入是“技术报修”还是“商务咨询”，从而让流程走向不同的处理分支。
  • 条件分支：传统 IF/ELSE。 场景：如果 AI 评分 > 80 分，则直接发送给客户；如果 < 80 分，则转发人工审核。
  • 迭代 (Iteration)：处理“一批”数据。 场景：给出一组竞品公司的官网链接，让 AI 挨个点进去抓取最新动态并汇总。
  • 循环 (Loop)：针对特定逻辑反复运行。 场景：AI 编写代码后自动测试，若不通过则携带错误日志返回重新修改，直到成功。
超级加工厂

3. 转换与处理节点

  • 代码执行：脚本处理复杂计算。 场景：获取当前日期并计算距离合同到期还有多少天。
  • 模板转换：自动套用格式。 场景：将获取到的“张三、25 岁”信息，自动填入《求职意向回复函》模板。
  • 变量聚合器/赋值：统合与记录。 场景：分别从搜索、数据库获取信息汇总；或记录“已处理段落数”。
  • 文档/参数提取器：精准提取非结构化信息。 场景：从乱糟糟的收货信息中，精准挖出“姓名”、“电话”、“地址”。
外部生态

4. 连接外界插件

  • Google / Tavily：实时联网。 场景：查询“今日最新的人民币汇率”，补齐模型知识滞后问题。
  • Markdown 转换：网页提效。 场景：将抓取到的乱码网页源代码，一键清理成层级分明的干净文档。
  • JSON 处理：系统间通信。 场景：将 AI 生成的非结构化文案转化成标准数据格式，直接录入 ERP。
  • HTTP 请求：万能插座。 场景：流程结束后，自动向飞书群机器人发送一条“任务已完成”的推送。

CHAPTER 02

实战应用：RAG & Workflow

知识库搭建 · 自动化工作流 · Prompt 工程

案例 1：为什么要搭建个人知识库

痛点直击：

“问卷中 29% 的同事提到‘信息分散’是最大痛点。”

  • 收藏文章找不到？
  • 文档散落无法统一检索？
  • 只能逐个打开 PDF 手动搜索？

原理与准备

核心原理：类似 AI 的“Google 搜索”。Top K 检索解决 Context Limit。

准备工作：
  1. 登录连山应用平台
  2. 创建新知识库 -> 上传文档 (PDF/Word/TXT)

核心配置：分段与检索调优

1. 分段 (Chunking)

新手推荐：父子模式。
定位“子句”的同时提供“父段落”，让 AI 不断章取义。

2. 检索 (Retrieval)

混合检索：语义 + 关键词。
Top K = 3, Score = 0.5。

案例 1：ChatFlow 构建与避坑

1. 构建步骤

  1. 工作室 -> 创建“个人知识助理”。
  2. 拖入“知识检索”节点并连接。

2. 核心避坑

  • Prompt 必须显式告知“参考检索内容”，否则 AI 会盲答。
  • K 值平衡：过大高延迟，过小漏真理。

3. 适用边界

✅ 适合：海量文档片段检索。
❌ 不适合：全局趋势分析或复杂统计（如“统计所有合同总金额”）。

  • 每天手动复制粘贴？

  • 想把“搬运工”的时间省下来？

第三步：结构提取 (LLM JSON)

  • 添加 LLM 节点。
  • Prompt：仅关注资讯列表，忽略广告。
  • 关键设置：开启结构化输出 (JSON)。

Prompt 工程：为什么要学？

核心认知：

RAG ChatFlow, Agent Workflow, 任何地方都离不开 Prompt。

大模型本质是一个概率预测引擎。

  • 语境密度越高，预测的“确定性空间”越窄。
  • 好 Prompt 能精准击中痛点。
  • 烂 Prompt 只能在平均值里打转。

Prompt 工程：“3+1”结构化公式

🎭 Role (角色)

“你是一位拥有 15 年经验的资深质量管控经理...”
赋予 AI 深度权限。

📝 Task (任务)

使用强动词：对比、提取、生成、校对。
明确必须完成的动作。

🚫 Constraints (约束)

字数、格式、禁语。
“严禁使用形容词”、“输出 JSON 格式”。

✨ Refine (追问)

好 Prompt 是调出来的。
通过“追问”微调直到完美。

实战对照：实习生 vs 专家

要素 ❌ 坏例子 (敷衍) ✅ 好例子 (精准)
Role “你是一个 AI，帮我写代码。” “你是一位精通 Dify 积木逻辑 的认证开发工程师。”
Task “把这两个文档总结一下重点。” “提取出 3 个潜在质量风险点并按紧急程度排序。”
Constraints “内容要全面，别漏了东西。” “使用表格输出结果，严禁使用形容词，字数 200 下。”

落地提效五大秘籍

1. 极度具体
明确受众，明确立场。
2. 给它例子
范本让 AI 模仿。
3. 强制分步
“请先思考核心逻辑...”
4. 定死格式
强制 JSON 实现流转。
5. 回归本质
是思路加速器而非搜索。
